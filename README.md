# 基于Java爬虫的微信小说阅读器

## 项目简介

这个项目是我毕业设计时做的，通过Java爬虫爬取网络上的小说，

## 计划清单

- 多个网站表与总表的连接
- 爬虫的下载程序希望能够

## 下一步的调整计划

- 目录表存储的目录用的是json串，但是如果有的小说章节太多，存储和查询就会消耗大量资源，每次更新都必须全部更新一次，而如果用mysql的json串会有较大的数据存不进去
- 小程序没有网站解析源的选择
- 单独更新某一本书
- 增加一个存储小说与网站解析的对应表，表中为空表示没有对应，有对应存入对应的URL链接，每个网站一个字段，每增加一个网站就要增加一个字段
